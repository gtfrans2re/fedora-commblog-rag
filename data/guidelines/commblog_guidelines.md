# Fedora Community Blog — Article Guidelines

> The Community Blog is one of the most important platforms for how information and news
> about the Fedora community, for the Fedora community, is shared. Following these
> guidelines helps keep the Community Blog organized, easy to navigate, and boosts
> search engine optimization (SEO) and accessibility of content.

---

## For Writers and Editors

### Content

- Put the important takeaway information in the first paragraph. Do not assume the
  reader will read the whole post.

### Formatting

- Remove all HTML tags from your post except for headings, bold/emphasis formatting,
  hyperlinks, and tables. WordPress does not show advanced HTML well.
- Insert a **"Read More" divider** after the first paragraph. This keeps the front page
  compact and helps bring readers to the Community Blog when content is reshared
  elsewhere.

### Images

- Images should have a relevant caption and alt tag.
- Provide proper attribution for images as required by the license.
- Upload images to WordPress. Do not hotlink to images hosted elsewhere.
- You may use as many images as you need, but be mindful of low-bandwidth readers.
- Do not include your featured image in the body of the post.

### Headers

- Use headers for longer articles (roughly 400 words or more).
- Use important keywords in headers — search engines use them to find relevant pages.
- Use lists where they improve readability. Use unordered lists unless sequence matters.
- Use blockquotes for long quotes.
- Do not use "here" as link text — this is bad for accessibility and SEO.

### Metadata

- **Category:** Add a single category. Events-related posts should have "Events" plus
  the topical category. Use "Fedora Project Community" sparingly. File a ticket with
  CommOps to request a new category.
- **Tags:** Add 3–6 tags per article. Avoid generic tags like "development". Prefer
  specific, strategic keywords like "Anaconda" or "Diversity and inclusion". Follow
  proper capitalization. Avoid creating new tags when possible.
- **Featured image (optional):** A featured image is recommended. Use
  [Unsplash](https://unsplash.com) for public domain images if needed. Do not include
  the featured image in the body of the post. Send new featured images as a pull request
  to the Fedora Community Blog images repo on Pagure. Use the article title in the
  featured image alt tag for an SEO boost.

## For Editors

- Ensure the author has a **Discourse username** set in their profile (their Fedora
  Account ID). This is necessary for the Discourse topic to have the correct username.
- If a post publishes under the "system" Discourse user, a Discourse admin can change
  the owner after the fact.

### Yoast SEO Plugin

The Yoast SEO plugin provides live feedback on articles and is the main plugin editors
will interact with.

- **Set a focus keyword.** Use one specific word or phrase — multiple keywords will
  throw off the algorithm. Not required for regular updates.
- **Set the article snippet.** This is the text shown when the article is posted to
  social media or other platforms.
- **Use the focus keyword in the snippet.** This is a significant SEO boost.
- Resolving red-bubble suggestions is helpful but not mandatory. The Community Blog
  targets contributors, not an external audience — get the big points right and ship.

### Scheduling

- Publish no more than **one article per day**. Multiple articles on the same day
  reduce total views for both.
- Aim for **Tuesdays and Thursdays** — these balance against Fedora Magazine, which
  publishes on Mondays, Wednesdays, and Fridays.
- Share content interesting to a wider audience with the **Social Media team** via the
  social-media mailing list. A subject line and a link is all you need.
- **Add the post to the Discourse calendar.** If you move a post, update the schedule
  date in WordPress and edit the corresponding Discourse schedule thread post.

### Theme / Design Issues

For issues with the Community Blog theme, use the Pagure repo for the theme.
**Ryan Lerch** is the designer and best point of contact for theme support.
